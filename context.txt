The content-management page is wired to real Supabase data through apps/admin-portal/src/lib/api.ts.
Edit Post actions are currently split between local UI state and persistence calls:
  - Save: persists via updateContent() or createContent()
  - Delete: persists via deleteContent() (soft delete by status="deleted")
  - Status switch: currently local-only in page state and does NOT persist to DB yet

Required behavior update:
  1) Add double confirmation for Save Changes
  2) Add double confirmation for status switch (pending/private/published)
  3) Upgrade Delete to double confirmation

1) FRONTEND MAPPING (CONTENT-MANAGEMENT)
============================================================

Primary files:
  - apps/admin-portal/src/app/dashboard/content-management/page.tsx
  - apps/admin-portal/src/components/edit-post-panel.tsx
  - apps/admin-portal/src/lib/api.ts

Current action paths:

1.1 Save Changes button
  Location:
    - edit-post-panel.tsx footer button "Save Changes"
  Flow:
    - handleSave() in EditPostPanel builds updated ContentItem
    - calls onSave(updated)
    - page.tsx handleSave callback:
      - if new item (no id): createContent(updated)
      - else: updateContent(updated)
      - updates local items[] state after success
  Current confirmation:
    - No confirmation modal before save

1.2 Status switch button (Published/Pending/Private)
  Location:
    - edit-post-panel.tsx footer button (Eye/EyeOff + current status label)
  Flow:
    - onToggleStatus(item.id)
    - page.tsx handleToggleStatus(id): cycles local status
      published -> pending -> private -> published
    - updates local items[] only
  Current confirmation:
    - No confirmation modal before status change
  Persistence gap:
    - updateContentStatus() is imported in page.tsx but not used
    - status change is currently NOT persisted to Supabase

1.3 Delete button
  Location:
    - edit-post-panel.tsx footer button "Delete"
  Flow:
    - click Delete -> sets confirmDelete=true
    - shows modal "Delete Post?"
    - confirm button calls onDelete(item.id)
    - page.tsx handleDelete(id) -> deleteContent(id)
    - deleteContent() performs soft delete in DB by updating status="deleted"
  Current confirmation:
    - Single confirmation modal exists
