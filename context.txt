2) COMPUTED STATE (useMemo BLOCKS)
============================================================

2.1 Filtered List
  const filtered = useMemo(() => {
    - Filters accounts by:
      1. Search query: brandName.includes(query) || contact.includes(query) || category.includes(query)
      2. Status filter: filterStatus === "all" || account.status === filterStatus
    - Re-computed whenever accounts[] or search or filterStatus changes.
    - Output: filtered array of matching accounts.
  }, [accounts, search, filterStatus])

2.2 Paginated Slice
  const paginated = useMemo(() => {
    - Slices filtered[] to display only current page's rows.
    - Calculation: start = (currentPage - 1) * rowsPerPage
    - Output: subset of filtered array for current page (e.g., rows 0–4 if page 1 and rowsPerPage=5).
  }, [filtered, currentPage, rowsPerPage])

2.3 Total Pages
  const totalPages = Math.max(1, Math.ceil(filtered.length / rowsPerPage))
    - Ensures totalPages is at least 1 (for empty results).
    - Used to disable "next page" button.

2.4 Pending Count
  const pendingCount = useMemo(() => 
    accounts.filter(account => account.status === "pending").length
  )
    - Displays "X pending verifications" badge above table.

2.5 Flagged Count
  const flaggedCount = useMemo(() => 
    accounts.filter(account => account.flagged).length
  )
    - Displays "X flagged account" badge above table.

2.6 Selected Detail Lookup
  const selectedDetails = selectedAccount ? accountModalDetails[selectedAccount.id] : null
    - Maps selectedAccount.id to hardcoded modal detail object (website, contactName, contactEmail, documentName).
    - accountModalDetails is a Record<string, {website, contactName, contactEmail, documentName}>.
    - IDs "1"–"5" have predefined static metadata.

3) SIDE EFFECTS & HANDLERS
============================================================

3.1 Reset Pagination
  useEffect(() => {
    setCurrentPage(1);
  }, [search, filterStatus, rowsPerPage])
    - Resets to page 1 whenever search, status filter, or rows-per-page changes.
    - Prevents user from being stuck on page 5 after applying a filter that only shows 2 rows.

3.2 Escape Key to Close Modal
  useEffect(() => {
    const onKeyDown = (event: KeyboardEvent) => {
      if (event.key === "Escape") {
        setSelectedAccount(null);
      }
    };
    window.addEventListener("keydown", onKeyDown);
    return () => window.removeEventListener("keydown", onKeyDown);
  }, [])
    - Adds global keyboard listener on mount.
    - Pressing Escape closes the detail modal (sets selectedAccount=null).

3.3 Update Account Status
  const updateSelectedAccountStatus = (nextStatus: UserAccount["status"]) => {
    if (!selectedAccount) return;
    
    // Update accounts array (immutable):
    setAccounts((prev) =>
      prev.map((account) =>
        account.id === selectedAccount.id ? { ...account, status: nextStatus } : account
      )
    );
    
    // Update selectedAccount so modal reflects new status:
    setSelectedAccount((prev) => (prev ? { ...prev, status: nextStatus } : prev));
  }
    - Invoked by "Approve & Notify" (→ verified) and "Reject" (→ suspended) buttons.
    - Updates both the accounts array AND the selectedAccount to reflect the change immediately in modal.
    - Changes do NOT persist to backend yet (TODO: hook up to updateUserStatus or backend API).

7) KEY UI PATTERNS & INTERACTIONS
============================================================

7.1 Search & Filter Flow
  User types in search → setSearch(value) → filtered recalculates → paginated recalculates → currentPage resets to 1.
  
  Result: Table updates to show only matching rows from page 1.

7.2 Pagination Flow
  User clicks next page → setCurrentPage(+1) → paginated recalculates → table shows new rows.
  
  Edge case: If user searches and result fits on 1 page, "next" button is disabled.

7.3 Modal Open/Close Flow
  User clicks row:
    1. onClick handler → setSelectedAccount(account)
    2. Modal appears because selectedAccount !== null
    3. selectedDetails = accountModalDetails[account.id] (lookup)
    4. Modal displays organization + document details
  
  User clicks Escape:
    1. Global onKeyDown handler fires
    2. setSelectedAccount(null)
    3. Modal disappears

  User clicks outside modal:
    1. Outer div onClick → setSelectedAccount(null)
    2. Modal disappears

7.4 Status Update Flow
  User clicks "Approve & Notify" or "Reject" button → updateSelectedAccountStatus(nextStatus):
    1. Immutably maps accounts array, changing selected account's status
    2. Immutably updates selectedAccount so modal reflects change immediately
    3. Both state updates happen synchronously
    4. Modal buttons remain functional; can click again to flip status back (currently)
